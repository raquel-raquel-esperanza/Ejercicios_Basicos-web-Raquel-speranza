01 — Fundamentos

¿QUÉ ES EL DISEÑO DE ARQUITECTURA?

La EL DISEÑO DE ARQUITECTURA Es la estructura organizacional de un sistema de software. Define cómo se dividen, organizan y comunican los componentes de una aplicación.

Es como el plano de un edificio: sin una buena arquitectura, el software se vuelve difícil de entender, mantener y escalar.

La arquitectura de software evolucionó desde instrucciones caóticas en los años 50 hasta sistemas distribuidos inteligentes en la actualidad, siempre buscando que el software sea más organizado, escalable y fácil de mantener.

🏢

02 — importante

Un buen diseño

Es el mapa completo del sistema que define lo que se construye, cómo se conecta y con qué tecnologías, garantizando que sea robusto, seguro y escalable desde el principio.

🔨

Principios de buen diseño - SOLID

S Single Responsibility Principle-Cada módulo hace una sola cosa.

O Open/Closed Principle-Abierto para extender, cerrado para modificar.

L Liskov Substitution Principle-Las subclases deben poder reemplazar a las padre.

I Interface Segregation Principle-Las subclases deben poder reemplazar a las padre.

D Dependency Inversion Principle-Depender de abstracciones, no de implementaciones

🧩

Otros principio

DRY: Don't Repeat Yourself-no duplicar código.

KISS: Keep It Simple—mantenerlo simple.

YAGNI: You Aren't Gonna Need It—no construir lo que no se necesita.

SoC: Separation of Concerns—separar responsabilidades

03 — Historia

Evolución del Diseño de Arquitecturade Software

2 Años 50 — Los inicios

No existía arquitectura formal

Los programas eran simples instrucciones secuenciales.

Se programaba directamente en lenguaje máquina y ensamblador.

No había separación de componentes.

Los programas cabían en tarjetas perforadas.

Años 60 — Primeras estructuras

Nace la programación estructurada

Se crean los primeros lenguajes de alto nivel: FORTRAN, COBOL, LISP.

Edsger Dijkstra propone eliminar el GOTO y estructurar el código.

Aparecen las primeras subrutinas y funciones.

Surge el concepto de modularidad.

Años 2020 — Arquitectura moderna

En la actualidad, se observa una evolución en la arquitectura de software con el auge de tecnologías como Serverless, Inteligencia Artificial y Edge Computing.

Se populariza Serverless (AWS Lambda, Azure Functions).

Aparece GraphQL como alternativa a REST.

Crece la arquitectura orientada a eventos.

Surge el Edge Computing (procesamiento cerca del usuario)

La Inteligencia Artificial se integra como componente arquitectónico.

Evolución resumida

1950s → Código espagueti (sin estructura).

1960s → Programación estructurada.

1970s → Modularidad y abstracción.

1980s → POO y arquitectura en capas.

1990s → Patrones de diseño y MVC.

2000s → SOA, REST y APIs.

2010s → Microservicios, Docker, Cloud.

2020s → Serverless, Edge, IA integrada.
